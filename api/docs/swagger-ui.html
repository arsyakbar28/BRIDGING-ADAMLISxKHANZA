Adam LIS x Khanza Bridging

Dokumentasi integrasi registrasi dan update hasil pemeriksaan antara SIMRS Khanza dan Adam LIS. Registrasi dapat memakai metode GET atau POST, sedangkan hasil pemeriksaan PK dikirim balik oleh Adam LIS dengan POST update-hasil.

Local Sandbox http://localhost:5005
PK Registrasi http://192.168.100.111:2311/adam-lis/bridging_sim_rs/registrasi
PA Registrasi http://192.168.100.111:2312/api/v1/adamlis/patologi-anatomi/bridging/registrasi
MK Registrasi http://192.168.100.111:2313/api/v2/adamlis/mikrobiologi/bridging/registrasi

Instalasi & Setup

  1. Pastikan Node.js, MySQL/MariaDB, dan database sik sudah tersedia.
  2. Install dependency dengan npm install.
  3. Sesuaikan koneksi database dan credential auth di .env.
  4. Jalankan server dengan npm start, lalu buka Swagger di /api-docs.

.env

                    
                        DB_PORT=3306
                        DB_NAME=sik
                        DB_USER=username
                        DB_PASSWORD=password
                        PORT=5005
GET Registration Adam LIS hit endpoint GET project ini untuk menarik order PK, PA, atau MK dari database Khanza.
POST Registration Project/SIMRS hit endpoint POST registrasi yang dibuat Adam LIS sesuai jenis lab.
Pilih Satu Untuk satu implementasi registrasi, gunakan GET atau POST. Jangan pakai keduanya untuk order yang sama.
PK POST Update Adam LIS hit POST /adam-lis/bridging/update-hasil untuk mengirim hasil dari analyzer ke SIMRS.
PA/MB Arsip Adam LIS hit endpoint arsip PA/MB untuk create hasil, lalu PUT arsip jika ada revisi.
Mapping Service Data disimpan ke tabel sesuai mapping dan service PK, PA, atau MK yang sudah dibuat.
Auth Lokal Endpoint lokal protected memakai Bearer token dari Auth, sedangkan POST registrasi eksternal mengikuti aturan Adam LIS.

Ringkasnya: registrasi pilih salah satu metode GET atau POST. Untuk PK, update hasil pemeriksaan dari LIS memakai POST update-hasil dengan response SIMRS yang wajib mengikuti format sukses/gagal Adam LIS.
